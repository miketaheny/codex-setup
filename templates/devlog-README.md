# Devlog

Detailed engineering history lives in this folder.

Use one Markdown file per meaningful commit. If a branch will be squashed before merge, use one file for the planned squash commit.

Suggested filename:

```text
YYYY-MM-DD-<commit-subject-slug>.md
```

Each entry should include:

- branch/worktree
- commit subject and commit SHA when already known
- goal
- files changed
- decisions made
- validation run
- review result
- follow-ups or known risks

Do not rewrite unrelated devlog files from other branches or worktrees.
